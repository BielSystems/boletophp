3. COMO CONTRIBUIR

   Você pode contribuir tanto com código como também melhorando a documentação
   do projeto. Ambos os casos o procedimento é o mesmo:
   
   3.1 Faça o fork do nosso repositório https://github.com/BielSystems/boletophp
       
   3.2 Download o seu repositório que você acabou de forkear com o seguinte
       comando:
       
       git clone git@github.com:USUARIO/boletophp.git  OU
       git clone https://USUARIO@github.com/USUARIO/boletophp.git
       
       onde USUARIO deverá ser substituido pelo seu nome de usuário no Github.
        
       Leia mais sobre como fazer forks em http://help.github.com/fork-a-repo
       
   3.3 Use o comando git checkout NOME-DO-BRANCH, Ex. git checkout 2.x-dev ,
       para acessar a versão de desenvolvimento que queira trabalhar.
       
   3.4 Faça as modificações no código ou documentação (arquivos README.txt).
       
       Lembrado que para modificar este arquivo README.txt você precisa fazer o
       checkout em master, Ex. git checkout master . Não crie ou coloque nenhum
       outro arquivo ou pasta, com exceção do README.txt, no branch master.
   
   3.4 Faça o(s) commit(s) de sua(s) modificação(ões) a medida que for fazendo
       modificações.
       
   3.5 Finalmente faça o pull request de sua(s) alteração(ões) para que um
       administrador do nosso repositório possa analisar e aprovar a(s) sua(s)
       modificação(ões).
       
       Leia mais sobre como fazer pull requests em
       http://help.github.com/send-pull-requests
